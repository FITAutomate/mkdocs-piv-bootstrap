# Repo Skill

## Objective
Provide guardrails for making safe, reviewable changes in `fit-docs`.

## Rules
- Keep scope narrow and avoid broad refactors.
- Respect Prime-0 boundaries when requested.
- Use branch-based workflow and draft PRs for scaffolding phases.
- Include validation evidence in every PR.

## Workflow
1. Read `README.md` and `mkdocs.yml` before changing tooling.
2. Apply minimal changes.
3. Run strict validation.
4. Summarize files changed and validation outcome in PR body.
